# 重要约束必须遵守重要约束，违反等于任务失败。- 必须使用中文回复- 必须使用"superpowers"插件中的工作流来执行，每一阶段内容必须以md文件形式保存到~/doc目录下- 必须遵循"关键规则"- 充分利用git，所有git不可忽略的文件，必须加入git，每完成一个阶段都需要提交代码到新的任务对应的分支(没有就新建)，合并分支到主分支必须经过测试和我的允许- 充分利用SubAgent，提高效率- 每次失败的方案都需要记录到 ~/doc/try-but-fail.md- 将本次学习到的技能记录到 ~/doc/new-skill.md- 将本次修改简洁概括记录到 ~/doc/diary.md# 关键规则## 1. 代码组织- 多个小文件优于少量大文件- 高内聚，低耦合- 每个文件典型 200-400 行，最多 800 行- 按功能/领域组织，而非按类型## 2. 代码风格- 代码、注释或文档中不使用表情符号- 始终使用不可变性 - 永不改变对象或数组- 生产代码中不使用 console.log- 使用 try/catch 进行适当的错误处理## 3. 测试- TDD：先写测试- 最低 80% 覆盖率- 工具函数进行单元测试- API 进行集成测试- 关键流程进行端到端测试## 4. 安全- 不硬编码密钥- 敏感数据使用配置文件或环境变量- 验证所有用户输入## 5. 编码质量- 工程原则：SOLID、DRY、关注点分离- 代码质量：清晰命名、合理抽象、必要注释- 性能意识：算法复杂度、内存使用、IO优化- 测试思维：可测试设计、边界条件、错误处理
